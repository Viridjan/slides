OS vs App

Il sistema operativo non è un programma come gli altri

🏗️

Sistema Operativo

Il fondamento: gestisce hardware, memoria, processi e periferiche. Esempi: Windows, macOS, Linux, Android.

Gira sempre in background, anche quando non si vede nulla sullo schermo.

📦

Applicazioni

Programmi che svolgono compiti specifici usando le risorse che l'OS mette a disposizione. Esempi: Word, Chrome, VLC, Spotify.

Non possono girare senza un OS sottostante.

🔄 Come collaborano

Quando avvii un'app, l'OS:

1 — La carica nella RAM
2 — Le assegna tempo CPU
3 — Gestisce l'accesso a disco e rete
4 — Controlla permessi e errori

L'utente vede solo l'app. L'OS è invisibile, ma presente.

💡 Analogia

L'OS è il palazzo. Le app sono gli inquilini. Il palazzo fornisce elettricità, acqua, ascensore — le app usano queste risorse senza preoccuparsene.

Installazione

Installare un programma: cosa succede davvero

1
Copia dei file — i file del programma vengono copiati nelle cartelle del sistema (es. C:\Program Files su Windows, /usr/bin su Linux).
2
Registrazione nel sistema — il programma viene "annunciato" all'OS: icone nel menu, associazioni file, servizi in avvio.
3
Librerie aggiuntive — se l'app ha dipendenze (DLL su Windows, shared libraries su Linux), vengono installate anch'esse.
4
Impostazioni di sistema — chiavi di registro (Windows) o file di configurazione scritti nelle cartelle utente.

Formati per OS

🪟 Windows

.exe (installer eseguibile) e .msi (Windows Installer). Esegui il file, segui la procedura guidata.

🍎 macOS

.pkg (installer guidato) e .dmg (immagine disco: trascini l'app nella cartella Applicazioni).

🐧 Linux — package manager

apt install, dnf install, pacman -S, zypper in: un comando, il sistema scarica e installa tutto in sicurezza dai repository ufficiali.

🤖 Android

Play Store (consigliato) oppure file .apk manuale (sideloading — più rischio).

Driver

Il ponte tra OS e hardware

Un driver è un software che traduce i comandi generici dell'OS in istruzioni specifiche per un componente hardware. Senza il driver corretto, l'hardware non funziona — o funziona in modo limitato.

⚙️ Perché aggiornarli

Un driver aggiornato può: migliorare le prestazioni (es. driver GPU per giochi nuovi), correggere errori e crash, aumentare la sicurezza (vulnerabilità nei driver sono bersagli reali), abilitare nuove funzionalità (es. nuova risoluzione).

⚠️ Attenzione

Scarica driver solo dal sito ufficiale del produttore (NVIDIA, AMD, Intel, HP…) o dal Windows Update. Siti terzi distribuiscono spesso malware mascherati da driver.

Esempi di periferiche

🖨️ Stampante — senza driver stampa solo testo grezzo o niente
🖥️ Scheda video — senza driver, risoluzione minima e no accelerazione 3D
🔊 Scheda audio — senza driver, nessun suono
📷 Webcam — senza driver, non rilevata da Teams/Zoom
🖱️ Tastiera/mouse speciali — tasti extra non funzionanti
📶 Adattatore WiFi — senza driver, nessuna rete wireless

💡 Su Linux

La maggior parte dei driver è già integrata nel kernel. Hardware comune funziona subito, senza installazione manuale — al contrario di Windows dove molti driver vanno cercati.

Aggiornamenti

Aggiornare OS e applicazioni: non rimandare

🖥️

Aggiornamenti del sistema operativo

Includono: patch di sicurezza (correggono vulnerabilità già sfruttate dagli attaccanti), bugfix (crash, freeze, errori), nuove funzionalità, supporto per hardware recente. Vanno installati appena disponibili — soprattutto quelli di sicurezza.

📦

Aggiornamenti delle applicazioni

Un OS aggiornato ma con app obsolete rimane vulnerabile. Browser, lettori PDF, plugin e suite Office sono bersagli frequenti. Le vulnerabilità nei browser sono tra le più sfruttate per attacchi.

🚨 WannaCry 2017

Il ransomware che ha infettato 200.000 sistemi in 150 paesi sfruttava una vulnerabilità di Windows per cui Microsoft aveva rilasciato la patch 60 giorni prima. Chi aveva aggiornato era immune. Chi no, ha perso tutti i file.

✅ Buone pratiche

• Abilita aggiornamenti automatici sull'OS
• Aggiorna le app tramite store ufficiale
• Non rimandare oltre 1 settimana le patch di sicurezza
• Verifica che i driver siano aggiornati, specie GPU e rete

Licenze software

Proprietario, Open Source, Software Libero

🔒

Software Proprietario

Codice sorgente segreto. Licenza d'uso a pagamento con condizioni definite dal produttore. Non puoi modificarlo né ridistribuirlo.

Esempi: Windows, macOS, Microsoft Office, Adobe Photoshop.

👁️

Open Source

Codice sorgente pubblico e leggibile. Focus sulla collaborazione tecnica: chiunque può contribuire, studiare, spesso modificare. Non sempre garantisce tutte le libertà.

Esempi: Android (AOSP), Chromium, VS Code.

🔓

Software Libero

Le 4 libertà (FSF):
0 — Eseguire
1 — Studiare
2 — Ridistribuire
3 — Modificare e ridistribuire

Focus sulla libertà dell'utente, non solo sul codice aperto. Gratis ≠ libero.

Esempi: Linux, Firefox, LibreOffice, VLC.

🔁 Come si sovrappongono

Quasi tutto il software libero è anche open source. Non tutto l'open source è "libero" nel senso politico (alcune licenze open source limitano usi commerciali). Molti software appartengono a entrambe le categorie: Linux, Firefox, Git, Python.

App Store

Gli store digitali: installazione sicura e centralizzata

🪟

Microsoft Store

Integrato in Windows 10/11. App verificate da Microsoft. Aggiornamenti automatici. Supporta anche app desktop tradizionali (Win32) dal 2021.

🍎

Apple App Store

Lo store più controllato: ogni app è revisionata da Apple. Sandboxing obbligatorio. Privacy label obbligatorie. Standard di sicurezza più alti.

🤖

Google Play Store

Milioni di app. Play Protect scansiona per malware. Meno restrittivo di Apple: qualche app malevola riesce a passare, ma viene rimossa velocemente.

🐧

Snap Store / Flathub

Store per Linux. Snap (Ubuntu/Canonical) e Flatpak/Flathub (community): app sandboxed e aggiornabili su qualsiasi distro, indipendentemente dal package manager.

✅ Vantaggi degli store

• App verificate (riduzione malware)
• Aggiornamenti automatici
• Disinstallazione pulita
• Un unico posto per cercare e gestire
• Rollback a versione precedente se un aggiornamento rompe qualcosa

⚠️ Fuori dagli store

Software non distribuito tramite store (es. software aziendale, tool professionali) va scaricato solo dal sito ufficiale del produttore. Mai da siti di download generici.

Account e Permessi

Utenti, privilegi e perché importano per la sicurezza

👑

Amministratore

Può fare tutto: installare software, modificare impostazioni di sistema, accedere a tutti i file. Su Windows: "Amministratore". Su Linux/macOS: root / sudo. Da non usare per il lavoro quotidiano.

🧑

Utente standard

Può usare le app installate, gestire i propri file, cambiare le proprie impostazioni. Non può installare software di sistema o modificare configurazioni globali. Consigliato per uso quotidiano.

👤

Ospite

Accesso minimo: usa il computer temporaneamente. Nessun file personale salvato tra una sessione e l'altra. Ideale per prestare il PC a qualcuno senza rischi.

🛡️ Principio del minimo privilegio

Ogni utente (e ogni app) dovrebbe avere solo i permessi strettamente necessari. Un malware che gira come utente standard fa molto meno danno di uno che gira come amministratore.

👨‍👩‍👧 Account separati

Su un PC condiviso (famiglia, ufficio) ogni persona dovrebbe avere il proprio account:

• Impostazioni personali separate
• Documenti accessibili solo a se stessi
• Password e sessioni distinte
• Nessuna contaminazione tra profili

📱 Su Android / iOS

Le app chiedono permessi espliciti (fotocamera, posizione, microfono, contatti). Concedili solo se l'app ne ha bisogno per funzionare. Un torcia-app non ha bisogno dei tuoi contatti.

Backup e Ripristino

Un backup è utile solo se si può ripristinare

💾

Cos'è un backup

Una copia dei propri dati in un luogo diverso dall'originale. Protegge da: guasti hardware, ransomware (i dati cifrati non lo sono nel backup), furto, cancellazione accidentale, errori software.

📍

Dove fare il backup

🔌 Disco esterno — veloce, economico, fisicamente separato
🏠 NAS (Network Attached Storage) — server casalingo automatico
☁️ Cloud — Google Drive, OneDrive, iCloud, Backblaze — accesso ovunque

La regola 3-2-1: 3 copie, su 2 supporti diversi, 1 fuori sede.

🔄

Ripristino

Recuperare documenti, impostazioni o l'intero sistema dal backup. Strumenti nativi:

🪟 Windows: Cronologia file, Punto di ripristino, Backup e ripristino
🍎 macOS: Time Machine (backup automatico ogni ora su disco esterno)
🤖 Android: Backup Google One
🐧 Linux: rsync, Déjà Dup, Timeshift

⚠️ Il backup non testato non esiste

Un backup non verificato potrebbe essere corrotto, incompleto o su un disco già guasto. Prova a ripristinare un file di test almeno una volta ogni 3 mesi. Il momento peggiore per scoprire che il backup non funziona è dopo un disastro.

Riconoscere l'OS

Come sapere quale sistema operativo usi

🪟

Windows

Metodo 1:
Start → Impostazioni → Sistema → Informazioni sul sistema

Metodo 2:
Tasto Win + Pausa/Break

Da terminale:
winver

🍎

macOS

Metodo 1:
Menu Apple (🍎) → Informazioni su questo Mac

Mostra versione (es. Ventura 13.6), chip (Intel o Apple Silicon), RAM e spazio disco.

Da terminale:
sw_vers

🐧

Linux

GUI:
Impostazioni → Informazioni (varia per distro)

Terminale:
uname -a — versione kernel
cat /etc/os-release — distro e versione
lsb_release -a — info complete

🤖

Android

Metodo:
Impostazioni → Informazioni sul telefono → Versione Android

Qui trovi anche:
• Livello patch sicurezza
• Versione kernel Linux
• Versione UI del produttore (es. One UI 6.1)

💡 A cosa serve saperlo

Sai se il tuo sistema riceve ancora aggiornamenti di sicurezza (es. Windows 10 termina supporto ottobre 2025). Puoi verificare la compatibilità con un nuovo software. Serve a chi ti supporta tecnicamente per darti istruzioni precise.

Riepilogo

I concetti fondamentali degli OS

📦 OS vs App

L'OS gestisce le risorse. Le app le usano. Senza OS, nessuna app può girare.

💿 Installazione

Ogni OS ha il suo metodo: .exe su Windows, dmg su Mac, package manager su Linux, APK su Android.

🔌 Driver

Traduttori tra OS e hardware. Tenerli aggiornati migliora prestazioni e sicurezza.

🔄 Aggiornamenti

OS e app vanno aggiornati regolarmente. Le patch di sicurezza non si rimandano.

📜 Licenze

Proprietario (chiuso), Open Source (codice aperto), Libero (4 libertà utente). Non sono la stessa cosa.

🏪 Store

Microsoft Store, App Store, Play Store, Flathub: installazione sicura, verificata e aggiornamenti automatici.

👤 Account

Account separati per ogni utente. Usa account standard per il lavoro quotidiano, non da amministratore.

💾 Backup

Regola 3-2-1: 3 copie, 2 supporti, 1 fuori sede. Testa il ripristino periodicamente.

🔍 Riconoscere l'OS

Impostazioni → Informazioni su tutti i sistemi. Sai se sei aggiornato e supportato.
