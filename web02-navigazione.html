Punto di partenza

Cosa significa
navigare
sul Web

Navigare significa richiedere e ricevere contenuti da server remoti tramite un programma chiamato browser. L'azione appare semplice; il processo sottostante è complesso.

🌐 Il browser

Il programma che usi per accedere al Web. Invia richieste ai server, riceve i dati (HTML, CSS, JS) e li trasforma in pagine leggibili.
Esempi: Chrome · Firefox · Safari · Edge · Opera

📋 Come si naviga

Digiti un indirizzo (URL) nella barra del browser, fai clic su un link, o usi una cronologia. Il browser fa tutto il resto: richiede, riceve, interpreta, mostra.

Come si parla in rete

I protocolli di rete

Un protocollo è un insieme di regole condivise che permettono a due sistemi diversi di comunicare. Come una lingua comune: se parlate la stessa, vi capite.

🗣️

Cos'è un protocollo

Un accordo su come formattare, trasmettere, ricevere e interpretare i messaggi. Definisce tutto: chi parla per primo, come si segnala un errore, come si chiude la connessione.

🌍

Protocolli del Web

HTTP/HTTPS — Scambio di pagine web
TCP/IP — Consegna dei pacchetti
DNS — Traduzione nome→IP
FTP — Trasferimento file
SMTP/IMAP — Email

🤝

Perché sono importanti

Grazie ai protocolli, un iPhone in Italia può comunicare con un server in Giappone. Senza standard condivisi, nessun sistema potrebbe parlare con un altro.

Analogia: il galateo è il protocollo di una conversazione educata — regole che tutti seguono per capirsi.

Il linguaggio del Web

HTTPHyperText
Transfer
Protocol

Il protocollo che definisce come browser e server si scambiano documenti ipertestuali (pagine web). Alla base di tutto il World Wide Web.

⚙️ Come funziona

Richiesta (Request): il browser chiede una pagina o risorsa.
Risposta (Response): il server risponde con i dati e un codice di stato (200 OK, 404 Not Found, 301 Redirect…)

⚠️ Limite di HTTP

I dati viaggiano in chiaro — non cifrati. Chiunque sia sulla stessa rete può leggerli. Inadatto per dati sensibili (password, dati bancari).

HTTP: testo leggibile da chiunque

Comunicazione sicura

HTTPSHTTP
Secure

La versione sicura di HTTP. I dati vengono cifrati tramite TLS (Transport Layer Security) prima di essere trasmessi. Anche se intercettati, risultano illeggibili.

🔒 Come funziona la cifratura

Browser e server negoziano una chiave segreta (handshake). Tutti i dati successivi vengono cifrati con quella chiave. Solo il destinatario può decifrarli.

🔑 Certificato digitale

HTTPS richiede un certificato SSL/TLS — un documento digitale che verifica l'identità del sito. Il lucchetto verde/grigio nel browser indica che il certificato è valido.

HTTPS: dati illeggibili in transito

Confronto

HTTP vs HTTPS

HTTP

🔓 Dati non cifrati — visibili in rete
⚡ Più veloce (nessuna cifratura)
❌ Non adatto a password, dati bancari
❌ Nessuna verifica identità del sito
📛 Browser mostra "Non sicuro"
🏛️ Usato ancora per contenuti pubblici non sensibili

HTTPS

🔒 Dati cifrati con TLS
🛡️ Protegge login, acquisti, messaggi
✅ Verifica l'identità del server (certificato)
🔑 Lucchetto nel browser
📈 Standard moderno — richiesto ovunque
🏆 Google favorisce HTTPS nel ranking

⚠️ Attenzione

HTTPS garantisce che la connessione sia cifrata, non che il sito sia affidabile. Un sito fraudolento può usare HTTPS. Controlla sempre anche il dominio.

✅ Regola pratica

Se devi inserire dati personali (username, password, carta di credito), verifica sempre il lucchetto e che l'indirizzo inizi con https://.

Identificatori del Web

Cos'è un URL

URL = Uniform Resource Locator. È l'indirizzo esatto di una risorsa sul Web. Come un indirizzo postale: identifica in modo univoco dove si trova qualcosa su Internet.

📍

Cosa identifica

Qualsiasi risorsa raggiungibile via Internet: una pagina HTML, un'immagine, un file PDF, un video, un file audio, un'applicazione web.

🌍

Univoco

Due URL diversi identificano due risorse diverse. Lo stesso URL punta sempre alla stessa risorsa (a meno che il server non la cambi o sposti).

🔗

Componenti

Protocollo, host (dominio), porta (opzionale), percorso, query string, frammento. Ogni parte ha un significato preciso che vedremo nella prossima slide.

https://www.wikipedia.org/wiki/Internet

Questo URL identifica la pagina "Internet" su Wikipedia in italiano

Struttura

Anatomia di un URL

https://www.sitofinto.edu.it/corsi/informatica?anno=2&tipo=base
Protocollo

https://

Definisce le regole di comunicazione. HTTP o HTTPS per il Web. FTP per file. mailto: per email. Termina sempre con ://

Host

www.sitofinto.edu.it

Il nome del server da contattare. Composto da sottodominio (www), nome (sitofinto), estensioni (.edu.it). Il DNS lo traduce in un indirizzo IP.

Percorso

/corsi/informatica

La "cartella" sul server dove si trova la risorsa. Simile al percorso di un file sul computer. Inizia sempre con /

Query

?anno=2&tipo=base

Parametri opzionali passati al server. Inizia con ?. Coppie chiave=valore separate da &. Usati per filtri, ricerche, opzioni.

Nomi della rete

Cos'è un
dominio?

Il dominio è il nome leggibile di un server su Internet. Gli indirizzi IP (come 192.168.1.1) sono numeri difficili da ricordare — i domini li rendono comprensibili.

🏷️ Come funziona

Il DNS (Domain Name System) traduce automaticamente il nome in un indirizzo IP. Digiti google.com → il DNS trova l'IP → il browser si connette.

📝 Struttura del dominio

www . wikipedia . org

www = sottodominio (opzionale)
wikipedia = nome del dominio (SLD)
.org = estensione di primo livello (TLD)

DNS: traduce nomi in indirizzi

Il tessuto del Web

Cos'è un hyperlink

Un hyperlink (o semplicemente link) è un elemento cliccabile che porta da una risorsa web a un'altra. È il concetto che ha ispirato il nome "World Wide Web".

🔗

Caratteristiche

Un link collega qualsiasi risorsa: pagina web, immagine, file da scaricare, email, numero di telefono, sezione della stessa pagina. Può essere testo, un'immagine, un pulsante.

<a href="url">testo</a>

🕸️

Il Web come rete di link

Il Web è una rete di documenti interconnessi tramite link. Tim Berners-Lee lo ha concepito nel 1989 come un sistema di ipertesti globale — ogni pagina può puntare a qualsiasi altra.

Miliardi di link formano il tessuto del Web moderno.

🔵 Link interni

Puntano a un'altra pagina o sezione dello stesso sito. Es: menu di navigazione, sommario di un articolo.

🌐 Link esterni

Puntano a un sito diverso. Es: una fonte citata, un social network, una risorsa esterna. Si aprono spesso in una nuova scheda.
