ARCHIVIAZIONE · M11
Il problema

Salvare non vuol dire capire

Quando premi "Salva", il computer deve decidere dove mettere i dati, come ritrovarli e chi può leggerli.

Il disco è il magazzino. Il file system è l'inventario. Le cartelle sono scaffali. I file sono gli oggetti.

📦

File

Un contenitore di dati: testo, immagine, video, programma, archivio.

🗂️

Cartella

Un modo per organizzare nomi e percorsi, non una scatola fisica.

🧭

Percorso

L'indirizzo che porta a un file dentro l'albero delle cartelle.

🧾

Metadati

Informazioni sul file: dimensione, date, permessi, proprietario.

Dischi magnetici

Hard Disk HDD

Un HDD usa piatti magnetici che girano e una testina che legge e scrive dati. È economico e capiente, ma ha parti meccaniche.

Idea chiave

È ottimo per grandi archivi: foto, video, backup. È meno veloce e più delicato negli urti rispetto a un SSD.

Piatti, tracce, settori e testina
ARCHIVIAZIONE · M11
Memoria flash

SSD: veloce, silenzioso, senza parti mobili

⚡

Veloce

Tempi di accesso bassi: sistema operativo e programmi si avviano più rapidamente.

🔇

Silenzioso

Non ci sono piatti che girano né testine da spostare.

🧮

Usura gestita

Le celle flash hanno cicli di scrittura limitati, distribuiti dal controller.

TRIM

Il comando TRIM informa l'SSD quali blocchi non servono più, aiutandolo a mantenere prestazioni e durata.

Prima del file system

Partizioni e formattazione

Un disco può essere diviso in partizioni: aree separate che il sistema vede come volumi distinti.

Formattare significa preparare una partizione con un file system. Di solito cancella l'indice dei file e rende lo spazio riutilizzabile.

Il catalogo

Cos'è un file system

È il metodo con cui il sistema operativo organizza nomi, cartelle, posizione dei dati, permessi e spazio libero.

🏷️

Nomi

Associa un nome leggibile a dati sparsi sul disco.

🧱

Blocchi

Divide lo spazio in unità gestibili: cluster o blocchi.

🔐

Regole

Può gestire permessi, quote, cifratura, snapshot e journaling.

Compatibilità

FAT32 ed exFAT

FormatoPunti fortiLimiti
FAT32Molto compatibile: PC, TV, console, dispositivi vecchi.File singoli fino a circa 4 GB; pochi strumenti moderni di sicurezza.
exFATBuono per chiavette e dischi esterni tra Windows, macOS e Linux moderni.Meno robusto di file system pensati per dischi di sistema.

Regola pratica: FAT32 per massima compatibilità vecchia; exFAT per trasferire file grandi su dispositivi esterni.

ARCHIVIAZIONE · M11
Windows

NTFS

È il file system tipico dei dischi di sistema Windows. Gestisce permessi, file grandi, journaling e funzioni avanzate.

È adatto a Windows, ma non è sempre la scelta migliore per chiavette da usare ovunque: alcuni sistemi lo leggono ma non lo scrivono senza supporto aggiuntivo.

🧾

Journaling

Tiene traccia delle operazioni per recuperare meglio dopo spegnimenti improvvisi.

🔐

ACL

Permessi dettagliati per utenti e gruppi.

📦

File grandi

Niente limite pratico da 4 GB come FAT32.

🪟

Windows

È la scelta naturale per il disco principale di un PC Windows.

ARCHIVIAZIONE · M11
Linux e macOS

Btrfs, ext4, APFS

Btrfs

File system moderno per Linux: snapshot, checksum, compressione trasparente, gestione avanzata dei volumi.

ext4

Molto diffuso su Linux: stabile, semplice, affidabile. Spesso è la scelta conservativa.

APFS

File system Apple moderno per macOS, ottimizzato per SSD, snapshot e cifratura.

Nota sul nome

Si scrive Btrfs, spesso pronunciato "butter FS". Non è "BRTFS", anche se l'errore è comune.

ARCHIVIAZIONE · M11
Metodo

Organizzare i file

Un buon archivio digitale non dipende solo dal disco: dipende da nomi, cartelle e regole coerenti.

Se non riesci a ritrovare un file, per te è quasi come se fosse perso.

1. Nomi chiari

2026-06-25_preventivo_rossi.pdf è meglio di nuovo2_finale_OK.pdf.

2. Cartelle per progetto

Raggruppa ciò che serve insieme: documenti, immagini, fatture, versioni.

3. Versioni controllate

Usa date o numeri di versione. Evita dieci file "finale".

ARCHIVIAZIONE · M11
Prestazioni

Frammentazione, spazio libero e salute

🧩

Frammentazione

Sugli HDD i pezzi di un file possono finire lontani. La testina deve muoversi di più.

🚫

Non deframmentare SSD

Gli SSD non hanno testine: deframmentarli è inutile e aggiunge scritture.

📉

Spazio libero

Tenere un po' di spazio libero aiuta aggiornamenti, cache, prestazioni e gestione dell'SSD.

Per la salute del disco controlla segnali SMART, errori frequenti, rumori strani sugli HDD e backup aggiornati.

ARCHIVIAZIONE · M11
Ridurre peso

File compressi

La compressione cerca ripetizioni e schemi nei dati per rappresentarli occupando meno spazio.

Due famiglie

Senza perdita: ZIP, PNG, FLAC. Puoi ricostruire l'originale.
Con perdita: JPEG, MP3, video compressi. Perdi dettagli per risparmiare spazio.

📝

Testo

Si comprime molto bene: contiene tante ripetizioni.

📷

Foto JPEG

Spesso è già compressa: zippandola guadagni poco.

🎬

Video

È già compresso con codec: ridurlo ancora può peggiorare la qualità.

📁

Cartelle

Un archivio può impacchettare molti file in un solo contenitore.

Scorciatoie evolute

Link simbolici e hard link

Un collegamento può far apparire un file in un posto diverso senza copiarne davvero il contenuto.

Link simbolico

È un file speciale che punta a un percorso. Se sposti o cancelli la destinazione, il link si rompe.

Hard link

È un secondo nome per lo stesso contenuto sullo stesso file system. Il dato resta finché esiste almeno un nome.

Collegamento Windows

I classici file .lnk sono scorciatoie gestite da Windows, simili come idea ma diverse tecnicamente.
